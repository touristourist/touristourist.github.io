安装Git:

  1. 下载Git安装包
  2. 将bin文件夹加入环境变量，即可在命令行中使用
  3. 配置git的user.name和user.email，将本地的公钥加入github账号。此步将git和github账号连接

安装node.js和npm:

  1. 下载安装包安装即可，若命令行不能使用，则配置环境变量

安装和使用Hexo:

  1. 首先使用npm install -g hexo-cli安装hexo模块，安装后会出现warn信息，不要紧，npm的node_modules中出现hexo-cli文件夹就说明安装完成。
  2. 若在命令行中还不能使用hexo命令，找到hexo-cli下的bin文件夹，加入环境变量即可
  3. 选择某个路径，使用命令hexo init [blogname]就会在该路径下创建blogname文件夹，blogname就是hexo博客主要操作的文件夹。
  4. 进入到blogname文件夹，使用命令hexo new [text_name]创建一篇名为text_name的markdown文件，hexo g生成，hexo s启动服务预览,之后在浏览器中的localhost:4000即可看到博客的本地预览。

平滑升级

[link]

总的来说就是用新的yml文件（路径为source/_data/xxx.yml，其中xxx为主题的名字）代替了原始的主题_config.yml文件。之后只有更改xxx.yml才会改变主题。这样的好处就是，如果对该主题用git pull更新，对于_config.yml文件里的配置可能会有更新，若之前用户是通过_config.yml文件改变主题配置，则在更新之后会引起未知的冲突或者覆盖用户之前的设置。而通过现在这种方法，即使更新对于xxx.yml文件也不会有影响，需要注意的是，若更新之后，_config.yml文件中加入了新字段，在xxx.yml文件中也应该要加上以保证和更新版本一致。
